# Image generation pipeline

This folder holds the tooling that produces the per-version item images and manifests consumed by
the gallery (repo root — `public/images/`).

## Pipelines

| Pipeline | Speed | Quality | Notes |
| --- | --- | --- | --- |
| **`standalone-mod/`** (recommended) | Fast | **Best** — direct render, true alpha, glass/translucent items | Fabric mod for MC **1.21.1**. `/itemgallery export` |
| `MinecraftItemPipeline.py` (legacy) | Slow | Original hotbar screenshots | `pyautogui` automation; fragile. Powers **v1 legacy** images on the site. |

Use the **standalone Fabric mod** for all new exports. It produces sharper, correctly transparent
icons and writes gallery-ready `manifest.json` / `hashes.json` / `changes.json` automatically.

## Recommended workflow

### 1. Build & install the mod

From [`standalone-mod/`](standalone-mod/):

```bash
./gradlew build
```

Install the jar from `build/libs/` plus [Fabric API](https://modrinth.com/mod/fabric-api) for **1.21.1**.
Join a world, then run:

```
/itemgallery export 1.21.1
/itemgallery export 1.21.1 64          # custom icon size (default 32)
/itemgallery export 1.21.1 64 all        # include modded namespaces
```

See [`standalone-mod/README.md`](standalone-mod/README.md) for `variants` and `dumptextures` modes.

Output lands in `<minecraft>/gallery-export/<version>/`:

```
acacia_boat.png, ...
manifest.json
hashes.json
changes.json    # when an older export folder exists for auto-diff
```

**Auto-diffing:** keep previous `gallery-export/<version>/` folders on disk so each new export
can diff against the last and write `changes.json`.

> Keep icon **scale** and **GUI scale** consistent across versions, or every icon may look "modified".

### 2. Package into the gallery

From the **repository root**:

**Legacy v1 set** (original site images — default):

```bash
npm run package-version -- --source "C:/Users/you/.minecraft/gallery-export" --mc-version 1.21.1
```

**Enhanced v2 set** (mod re-renders — recommended for new work):

```bash
npm run package-version -- --source "C:/Users/you/.minecraft/gallery-export" --mc-version 1.21.1 --set v2
```

| Flag | Result |
| --- | --- |
| *(default)* | `public/images/` + `versions.json` — legacy captures |
| `--set v2` | `public/images-v2/` + `versions.json` — enhanced mod captures |

The site **Images** selector switches between legacy (v1) and enhanced (v2). Each mode uses only that set’s versions and PNGs — package each re-export under `images-v2/` with `--set v2`.

### 3. Migrating from the Python pipeline

Mod renders are **not pixel-identical** to old screenshots, so don't diff a mod export against a
Python-era folder. To migrate cleanly:

1. Re-export the latest existing version with the mod (`--set v2` is fine).
2. Export each newer version from there, keeping all `gallery-export/` folders together.
3. Eventually re-export older versions into v2 as you have time — v1 stays available as legacy.

## File-name contract

Plain item file names: `stone.png`, `acacia_boat.png`. Modded items use `namespace__path.png`.
